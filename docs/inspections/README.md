# Inspection Reports

This folder collects inspection reports for the integration roadmap. Use the naming convention and template below so
future reviews can quickly compare scope, findings, and follow-up work.

## Naming convention

- **Format:** `YYYY-MM-DD-<topic>-inspection.md`
- **Examples:**
  - `2024-06-01-settings-persistence-inspection.md`
  - `2024-06-01-translation-entrypoints-inspection.md`

## Required sections

Every inspection report must include the following sections in this order:

1. **Scope**
   - What was inspected and why.
   - Out-of-scope items.

2. **Findings**
   - File paths, classes, methods, and risks.
   - Notes about uncertainty or conflicts.

3. **Next steps**
   - Follow-up tasks or validation steps.
   - Links to issue drafts when available.

## Template

```markdown
# <Title>

## Scope
- ...

## Findings
- ...

## Next steps
- ...
```
